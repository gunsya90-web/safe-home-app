# 대구 아파트 정보 가져오기 (아파트명·동·호)

SAFE-HOME 관리자 화면의 "🏢 아파트 정보 일괄 등록 (CSV)"에 붙여넣을 `아파트명,동,호` 목록을
공공데이터포털 API로 만들어주는 스크립트입니다. **평면도·소방시설현황은 포함하지 않습니다** —
그건 등록 후 관리자 화면에서 건물별로 발급하는 "소방시설 점검 링크"로 따로 채웁니다.

## 이 스크립트가 쓰는 데이터 (3단계)

| 단계 | 데이터셋 | 용도 |
|---|---|---|
| 1 | [국토교통부_공동주택 단지 목록제공 서비스](https://www.data.go.kr/data/15057332/openapi.do) (`getSigunguAptList3`) | 구/군 안의 등록 단지코드(kaptCode)·단지명(kaptName) 목록 |
| 2 | [국토교통부_공동주택 기본 정보제공 서비스](https://www.data.go.kr/data/15058453/openapi.do) (`getAphusBassInfoV4`) | 그 단지코드의 정확한 주소(도로명주소 doroJuso/지번주소 kaptAddr)와 법정동코드 |
| 3 | 국토교통부_건축물대장정보 서비스 — 전유부(`getBrExposInfo`) | 그 주소의 실제 동명칭(dongNm)·호명칭(hoNm) 목록 |

세 API 모두 이 스크립트가 `requests`로 직접 호출합니다(1·2단계는 JSON, 3단계는 XML 응답이라
`xmltodict`로 파싱). 처음에는 오픈소스 라이브러리(PublicDataReader)를 썼는데, 그 라이브러리의
전유부 조회 함수에 타임아웃이 없어서 서버 응답이 늦으면 스크립트가 무한정 멈추는 문제가 있어
직접 호출로 바꿨습니다 — 모든 요청에 15초 타임아웃이 걸려 있어 이제 절대 무한정 멈추지 않습니다.

## 직접 하셔야 하는 것 (자동화할 수 없는 부분)

1. [data.go.kr](https://www.data.go.kr) 회원가입
2. 아래 세 서비스를 각각 **활용신청** → 보통 자동승인(몇 분~1~2시간)
   - [국토교통부_공동주택 단지 목록제공 서비스](https://www.data.go.kr/data/15057332/openapi.do)
   - [국토교통부_공동주택 기본 정보제공 서비스](https://www.data.go.kr/data/15058453/openapi.do)
   - "국토교통부_건축물대장정보 서비스"(또는 건축HUB 버전)
3. 마이페이지에서 서비스키(인증키) 복사 — 세 서비스가 같은 계정으로 승인되면 서비스키는 보통 하나로 공용입니다.

이 세 가지는 본인 계정으로만 할 수 있어서 대신 해드릴 수 없습니다.

## 실행 방법

```bash
pip install pandas requests xmltodict
```

서비스키는 **파일에 적지 말고** 실행 직전에 환경변수로만 넘깁니다(이유는 아래 "보안 주의사항" 참고):

```bash
# Windows PowerShell (반드시 "PowerShell" 앱에서 — Python 인터프리터(>>>)나 Colab 아님)
$env:DAEGU_API_SERVICE_KEY = "발급받은_서비스키"
python fetch_daegu_buildings.py

# macOS/Linux
export DAEGU_API_SERVICE_KEY="발급받은_서비스키"
python fetch_daegu_buildings.py
```

필요하면 `fetch_daegu_buildings.py`의 `SIGUNGU_CODES`에서 원하는 구/군만 남겨도 됩니다.

끝나면 `daegu_buildings.csv`가 생깁니다. 이 파일을 열어 내용을 복사해서
관리자 화면(`?role=admin`)의 "아파트 정보 일괄 등록" 칸에 붙여넣고 "가져오기"를 누르면 됩니다.

## 보안 주의사항

- 서비스키는 **절대 이 저장소의 어떤 파일에도 직접 적어서 커밋하지 마세요.** 이 저장소는
  GitHub Pages로 배포되는 공개 저장소라, 커밋되는 순간 인터넷 누구나 볼 수 있습니다.
- 위 환경변수 방식으로 실행하면 키가 어떤 파일에도 남지 않습니다. 결과물인
  `daegu_buildings.csv`에도 키는 포함되지 않으니 그 파일만 안심하고 관리자 화면에 붙여넣으면 됩니다.
- 서비스키를 이미 어딘가(채팅, 메모, 캡처 이미지, 노트북 셀 등)에 붙여넣었거나 노출한 적이 있다면,
  data.go.kr 마이페이지에서 재발급(기존 키 폐기)하는 걸 권장합니다.

## 알아두면 좋은 점

- **진행 상황이 항상 로그에 찍힙니다** — 단지 목록 페이지, 각 단지 처리 시작(`[N/전체] OO 처리 중...`),
  성공(`✓ OO: N세대`) 등. 한참 동안 아무 로그도 안 뜨면 뭔가 잘못된 것이니 `Ctrl+C`로 멈추고 알려주세요.
- **모든 요청에 15초 타임아웃 + 최대 3번 재시도(2초→5초→10초 대기)**가 걸려 있습니다. 순간적인
  호출 제한(rate limit)이나 서버 지연은 자동으로 넘어가지만, 그래도 안 되면 그 단지만 건너뛰고
  계속 진행합니다(전체가 멈추지 않고, 실행 끝에 "건너뛴 단지 N개"로 알려줍니다).
- **전유부 API는 한 번에 최대 100세대까지만 줍니다.** 100세대가 넘는 단지는 페이지를 넘겨가며
  자동으로 이어받도록 되어 있습니다 — 만약 실행 결과에서 유독 여러 단지가 정확히 "100세대"로만
  나온다면 이 페이지네이션이 제대로 안 된 옛날 버전일 수 있으니 스크립트를 다시 받아 실행해주세요.
- 단지 수가 많으면 전체 실행에 시간이 꽤 걸릴 수 있습니다(단지당 최대 2번의 API 호출 + 최소
  2초 대기이므로, 대구 전체를 돌면 수십 분 이상 걸릴 수 있습니다).
- 응답 필드명은 대부분 Swagger 문서 또는 API 설명에서 확인한 값이지만, 혹시 "예상과 다른 응답"이나
  "필드를 찾지 못했습니다" 로그가 뜨면 함께 출력되는 실제 원문/필드 목록을 보고 알려주시면 바로 고칠 수 있습니다.
