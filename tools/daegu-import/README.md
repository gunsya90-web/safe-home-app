# 대구 아파트 정보 가져오기 (아파트명·동·호)

SAFE-HOME 관리자 화면의 "🏢 아파트 정보 일괄 등록 (CSV)"에 붙여넣을 `아파트명,동,호` 목록을
공공데이터포털 API로 만들어주는 스크립트입니다. **평면도·소방시설현황은 포함하지 않습니다** —
그건 등록 후 관리자 화면에서 건물별로 발급하는 "소방시설 점검 링크"로 따로 채웁니다.

## 이 스크립트가 쓰는 데이터

| 단계 | 데이터셋 | 용도 |
|---|---|---|
| 1 | [국토교통부_공동주택 단지 목록제공 서비스](https://www.data.go.kr/data/15057332/openapi.do) (`getSigunguAptList3`) | 구/군 안의 등록 단지명(kaptName)·법정동코드(bjdCode)·주소(as1~as4) 목록 |
| 2 | [국토교통부 건축물대장정보 서비스 — 전유부](https://www.data.go.kr/dataset/15004825/fileData.do) | 그 단지 주소의 실제 동명칭(dongNm)·호명칭(hoNm) 목록 |

1단계는 Swagger 문서에서 실제 응답 스키마를 직접 확인해 정확한 필드명으로 작성했습니다.
2단계는 [PublicDataReader](https://github.com/WooilJeong/PublicDataReader)라는 오픈소스
파이썬 라이브러리로 조회합니다(직접 만든 게 아니라 이미 검증된 라이브러리를 재사용).

## 직접 하셔야 하는 것 (자동화할 수 없는 부분)

1. [data.go.kr](https://www.data.go.kr) 회원가입
2. 아래 두 서비스를 각각 **활용신청** → 보통 1~2시간 내 자동 승인
   - [국토교통부_공동주택 단지 목록제공 서비스](https://www.data.go.kr/data/15057332/openapi.do)
   - "국토교통부_건축물대장정보 서비스"(또는 건축HUB 버전)
3. 마이페이지에서 서비스키(인증키) 복사 — 두 서비스가 같은 계정으로 승인되면 서비스키는 보통 하나로 공용입니다.

이 세 가지는 본인 계정으로만 할 수 있어서 대신 해드릴 수 없습니다.

## 실행 방법

```bash
pip install PublicDataReader pandas requests
```

서비스키는 **파일에 적지 말고** 실행 직전에 환경변수로만 넘깁니다(이유는 아래 "보안 주의사항" 참고):

```bash
# Windows PowerShell
$env:DAEGU_API_SERVICE_KEY = "발급받은_서비스키"
python fetch_daegu_buildings.py

# macOS/Linux
export DAEGU_API_SERVICE_KEY="발급받은_서비스키"
python fetch_daegu_buildings.py
```

필요하면 `fetch_daegu_buildings.py`의 `SIGUNGU_CODES`에서 원하는 구/군만 남겨도 됩니다.

끝나면 `daegu_buildings.csv`가 생깁니다. 이 파일을 열어 내용을 복사해서
관리자 화면(`?role=admin`)의 "아파트 정보 일괄 등록" 칸에 붙여넣고 "가져오기"를 누르면 됩니다.

## 보안 주의사항

- 서비스키는 **절대 이 저장소의 어떤 파일에도 직접 적어서 커밋하지 마세요.** 이 저장소는
  GitHub Pages로 배포되는 공개 저장소라, 커밋되는 순간 인터넷 누구나 볼 수 있습니다.
- 위 환경변수 방식으로 실행하면 키가 어떤 파일에도 남지 않습니다. 결과물인
  `daegu_buildings.csv`에도 키는 포함되지 않으니 그 파일만 안심하고 관리자 화면에 붙여넣으면 됩니다.
- 서비스키를 이미 어딘가(채팅, 메모, 캡처 이미지 등)에 붙여넣었거나 노출한 적이 있다면,
  data.go.kr 마이페이지에서 재발급(기존 키 폐기)하는 걸 권장합니다.

## 주의사항

- 건축물대장 "전유부" API는 지번(번지) 단위로만 조회되는 구조입니다. 1단계(단지 목록)의
  주소 필드(`as4`)에서 번지를 최대한 뽑아 2단계 조회에 쓰는데, 주소 표기 방식이 예상과
  다르면 그 단지는 건너뛰고 계속 진행합니다(전체가 멈추지 않고, 실행 끝에 "건너뛴 단지 N개"로
  알려줍니다) — 그래서 100% 완전하진 않아도 실행할 때마다 실제 데이터가 쌓입니다.
- 단지 수가 많으면 시간이 걸리고, 공공데이터포털의 초당 호출 제한에 걸릴 수 있어 스크립트에
  짧은 대기시간을 넣어뒀습니다.
- 전유부 조회 쪽 컬럼명(`dongNm`/`hoNm`)은 확인된 값이지만, 혹시라도 "전유부 컬럼을 찾지
  못했습니다"라는 로그가 뜨면 함께 출력되는 실제 컬럼 목록을 보고 스크립트의 `find_col` 호출
  부분 키워드만 조정하면 됩니다.
